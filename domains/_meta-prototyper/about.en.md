# Lab D-ND — meta-lab

## About

The meta-lab is the first D-ND lab operating at the system level, not at
the level of a specific domain. Its task is to **produce cognitive seeds**
for new laboratories: given a request on a target domain ("I want a lab
on X"), the meta-lab applies the D-ND modus to the domain itself and
extracts the first structured tension + the invariants the domain
possesses + the empirical verification feasible with available compute.

The output is not "a working lab" ready to use. It is a **seed**: the
precise formulation of the new lab's initial question, its natural
dipolar tensions, the model axioms projecting into the domain, the
numerically verifiable assertions. From this seed, the fixed D-ND_LAB
scaffolding (folder structure, base importers) creates the complete
file system, and the new lab's first cycle follows.

The meta-lab applies to the generated template the same filter that
domain labs apply to their own findings: an asymmetric counter-pole
with five lenses. M1 verifies that proposed tensions have arithmetic
dipoles (det≠0, not description). M2 that assertions are truly
executable (PASS/FAIL/SKIP, not print). M3 that initial tools run
out-of-box. M4 that a naive baseline exists against which to measure
the modus. M5 that the first cycle can produce new information, not
just restate.

If any of the five fails, the template is not installed: the meta-lab
declares the domain "not of leverage" and the refusal enters the
meta-lab cimitero as useful crystallization, not failure. This is A2
— the necessary boundary.

The meta-lab is in trial. Its first test is the regeneration of the
existing physics lab: given the system's internal corpus, the meta-lab
must produce a physics seed structurally equivalent to the original.
Zero diff on cognitive seed + equivalent cycle = meta-lab valid.
